# Developer Agent Research Policy

Agent-Reach may be used only for external technical research.

Allowed:

- public documentation
- public GitHub issues
- public blog posts
- public API docs
- public troubleshooting references

Not allowed:

- production secrets
- employee data
- uploaded bills
- private Teams messages
- Telegram chat contents
- Supabase service role keys
- OpenAI keys
- webhook keys

Research findings must be summarized before implementation.
Agent-Reach findings are advisory, not production source of truth.

Agent-Reach must NOT be added to:

- backend runtime
- frontend runtime
- production workflows
- user-facing features
- app business logic
